# Open Yukkuri Editor — 全体アーキテクチャ設計書

## 1. システム概要

Open Yukkuri Editor は、以下の3つのソフトウェアの機能を統合した次世代動画編集プラットフォームを目指します：

- **ゆっくりMovieMaker4 (YMM4)** — 音声合成＋タイムライン編集
- **AviUtl** — フィルタベース編集＋スクリプト制御
- **独自拡張** — モダンなプラグインシステム＋Web技術

## 3. コアモジュール設計

### 3.1 Timeline Engine
- **責務**: トラック・クリップの管理、再生ヘッド制御
- **既存**: 実装済み（`src/models/timeline.ts`）
- **拡張**: 多重トラック、グループ化、入れ子タイムライン

### 3.2 Scene Manager
- **責務**: プロジェクトをシーン（チャプター）に分割管理
- **YMM4互換**: `.ymmp` の scene タグに対応
- **機能**: シーン間トランジション、シーンテンプレート

### 3.3 Keyframe System
- **責務**: 全プロパティに対してキーフレームアニメーション
- **既存**: `models/timeline.ts` に `Keyframe` インターフェース定義済み
- **拡張**: 
  - イージング関数（Linear, EaseIn, EaseOut, Cubic, Elastic, Bounce）
  - ベジェ曲線エディター
  - プロパティごとの独立したキーフレーム

### 3.4 Effect Graph
- **責務**: 映像/音声エフェクトのDAG（有向非巡回グラフ）処理
- **設計**: 
  - 各クリップにエフェクトチェーンを持つ
  - エフェクトはプラグインとして動的ロード
  - プリセット保存/読み込み対応

### 3.5 Audio Engine
- **責務**: 音声再生、ミキシング、エフェクト
- **技術**: Web Audio API + AudioWorklet
- **機能**: 
  - 音量フェード、パンニング
  - ノイズ除去、コンプレッサー
  - 音声合成エンジン連携（VOICEVOX等）

### 3.6 Mask System
- **責務**: 図形/画像マスクによる部分表示制御
- **YMM4互換**: 標準マスク（矩形、円形、画像）
- **拡張**: アルファチャンネル、リニアマット、クロマキー

### 3.7 Script Engine
- **責務**: AviUtl互換スクリプト（.anm/.obj）の実行
- **設計**: TypeScriptベースのDSL、またはWebAssemblyで既存スクリプト実行
- **代替**: JavaScriptサブセットによる独自スクリプト言語

### 3.8 Renderer
- **責務**: Canvas/WebGLへの最終描画
- **現状**: HTML5 Canvas 2D（タイムライン）
- **拡張**: WebGL2/WebGPU によるGPUレンダリング
  - ピクセルシェーダーエフェクト
  - リアルタイムプレビュー
  - マルチレイヤー合成

## 4. データフロー

```
入力ファイル → パーサー → 内部モデル → エフェクトグラフ → レンダラー → プレビュー/出力
                ↑
            .ymmpファイル
                ↓
            シリアライザー → 保存
```

## 5. 技術選定理由

| 技術 | 用途 | 理由 |
|------|------|------|
| Electron | デスクトップシェル | クロスプラットフォーム、ネイティブAPI |
| React | UIフレームワーク | コンポーネント設計、エコシステム |
| TypeScript | 型安全性 | 大規模開発に必須 |
| Canvas 2D/WebGL | レンダリング | プラグイン表現力 |
| Web Audio API | 音声処理 | ノンブロッキング、高機能 |
| ffmpeg.wasm | エンコード出力 | ブラウザ内で完結 |
| Web Worker | プラグイン分離 | メインスレッド保護 |
